# Principles

- **Performant**: Every optimization — no matter how minor — matters. Older phones and tablets are the benchmark, not high-end desktops.
- **Native**: Looks and behaves like a built-in Obsidian plugin — consistent with platform conventions, not fighting them.
- **Polished**: Consistent spacing, visual coherence, and smooth transitions make the plugin a joy to use.
- **Depth over breadth**: Fix bugs before adding features. Do fewer things well rather than many things poorly.
- **User control**: Nearly everything is configurable through plugin settings, view options, Style Settings, and custom CSS. Power users can customize deeply; casual users get good defaults.
